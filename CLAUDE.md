# Emend ŌĆö Agent Context

A quiet, native **macOS (Apple Silicon)** Markdown editor ŌĆö a cogito.md clone (no Typefully).
**Rust core + Swift/SwiftUI UI.** Notes are plain files on disk (no DB/sync). AI is BYOM (OpenAI-compatible).

## Structure

- `crates/emend-core` ŌĆö ALL logic, **no FFI dep**, `cargo test` in isolation.
- `crates/emend-ffi` ŌĆö thin `#[uniffi::export]` shim (the only `uniffi` consumer).
- `swift/EmendCore` ŌĆö SwiftPM package wrapping the XCFramework + generated bindings.
- `app/Emend` ŌĆö Xcode macOS app (created during implementation).

## Project guardrails (do not violate)

- **No panics across FFI** (NFR-003): `emend-core` denies `unwrap_used`/`expect_used`/`panic`; fallible boundary calls return `Result<_, EmendError>`. UniFFI `catch_unwind` contains the rest.
- **FFI ranges are UTF-16 code units** (research ┬¦A2) to map onto `NSRange` ŌĆö never pass UTF-8 offsets across the boundary.
- **Swift owns the text buffer**; per-keystroke edits go to Rust as tiny deltas. Hot path is **synchronous**; async only for AI + search (cancellable via Rust handles, not Swift `Task`).
- **Privacy** (SC-008/FR-035): zero outbound network unless AI is configured AND invoked. Preview WebView blocks remote loads (CSP + nonPersistent + navigation delegate).
- **Autosave is atomic + durable** (FR-009a): tempfile ŌåÆ fsync ŌåÆ rename ŌåÆ fsync dir; debounced (don't fsync per keystroke). Feed post-write `(mtime,len)` to the watcher's self-write suppression so saves don't echo (FR-006a).
- **AI key**: Keychain only, transient to Rust, redacted in the HTTP client ŌĆö never logged/persisted (NFR-006).
- **Two Markdown engines on purpose**: tree-sitter (editor, incremental, advisory highlight) vs comrak (preview, authoritative HTML). Don't unify them.
- Conventional Commits required (DS-007); `lefthook` runs fmt/clippy/swift-lint pre-commit.
- **MSRV is 1.85** ŌĆö pin every new dep to a version whose `rust-version Ōēż 1.85` (CI enforces with `cargo +1.85 check --all`; e.g. `criterion` stays 0.7 because 0.8 needs 1.86, `ropey` 1.6). Never bump from memory.
- **Generated FFI is git-ignored build output**: `swift/EmendCore/Sources/EmendCoreFFI/` + `xcframework/` are regenerated by `just xcframework` (and in CI) ŌĆö run it before building the app. The generated `EmendCoreFFI` target compiles in **Swift 5 language mode** (its UniFFI foreign-trait VTables aren't Swift-6 strict-concurrency clean); `EmendCore` + the app stay Swift 6. Bindgen: `cargo install uniffi --features cli` (match the pinned uniffi). Foreign-trait sinks use `#[uniffi::export(with_foreign)]`; the Swift error is `FfiError` (exhaustive mirror of `EmendError`).
- **App unit tests are app-hosted** (`@testable import Emend`); headless logic (bookmark resolution, attribute mapping) is tested there. Security-scoped-bookmark tests use **plain** bookmarks (the test process isn't sandboxed).
- **No XCUITest / GUI automation**: a `bundle.ui-testing` runner can't bootstrap under CI's `CODE_SIGNING_ALLOWED=NO` (CI is GUI/signing-free by design). Test editor flows **headlessly** in the app-hosted bundle by driving `EditorCoordinator` + the pure transforms directly (see `EditorPersistenceTests` for the FR-009 round-trip), never via a launched GUI.
- **SwiftFormat and SwiftLint `--strict` can fight**: `swiftformat` wraps a long multi-line `if`/`guard` condition and drops `{` onto its own line, which SwiftLint `opening_brace` then rejects. Keep conditions one line ŌĆö precompute a boolean `let`, or use `guard ŌĆ” else { ŌĆ” }` (its brace follows `else`). `nesting` allows types Ōēż1 level deep; no force-unwraps (`force_unwrapping` is opt-in). Run `swiftformat app swift` then `swiftlint lint --strict` **before** the slow `xcodebuild`.
- **Editor edit model**: list/formatting behaviour is pure `(NSString, NSRange) ŌåÆ Edit` transforms (`SmartLists`/`FormattingCommands`, headless-tested); the `MarkdownTextView` applies each `Edit` via `shouldChangeText`/`replaceCharacters`/`didChangeText` so it registers undo **and** reaches the core through the same storage-delegate delta path as typing. A subclassed `NSTextView` needs an explicit TextKit 2 stack (`NSTextContentStorage`ŌåÆ`NSTextLayoutManager`ŌåÆ`NSTextContainer`); `scrollableTextView()` can't be subclassed.
- **NSOutlineView/AppKit in SwiftUI**: wrap in `NSViewRepresentable` with a `@MainActor` Coordinator that conforms to `NSOutlineViewDataSource`/`Delegate`/`NSMenuDelegate` ŌĆö those SDK protocols **are** `@MainActor`, so plain conformance compiles (unlike `NSTextStorageDelegate`, which needs `nonisolated`+`assumeIsolated`). Cross-thread RustŌåÆSwift callbacks (e.g. the watcher's `DocObserver`) bridge through a `final class` holding an immutable `@Sendable` closure that hops via `Task { @MainActor in ŌĆ” }`. As Swift files grow, SwiftLint enforces `type_body_length` (250 ŌåÆ move private helpers to a same-file extension) and `file_length` (400 ŌåÆ split the type into its own file).
- **US2 app-state is Swift-side**: the FFI exposes **no getter** for folder-icon/`isFavorite`/`isPinned`, and the core holds favorites/pins/icons/child-order in memory only (no persistence layer yet, despite the data-model's "core-owned store"). So `WorkspaceModel.AppState` (UserDefaults) is the display source of truth + persistence, forwarded to the core via setters and replayed on launch. Location bookmarks (security-scoped) likewise persist Swift-side. A future task should move this core-side + add the getters.
