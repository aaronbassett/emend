# Emend — Agent Context

A quiet, native **macOS (Apple Silicon)** Markdown editor — a cogito.md clone (no Typefully).
**Rust core + Swift/SwiftUI UI.** Notes are plain files on disk (no DB/sync). AI is BYOM (OpenAI-compatible).

**Governance**: [`.sdd/memory/constitution.md`](.sdd/memory/constitution.md) v1.0.0 — read it; Principles I–VII are binding.
Specs of record: `specs/001-markdown-editor/` — [spec](specs/001-markdown-editor/spec.md) · [plan](specs/001-markdown-editor/plan.md) · [research](specs/001-markdown-editor/research.md) · [data-model](specs/001-markdown-editor/data-model.md) · [FFI contract](specs/001-markdown-editor/contracts/ffi-interface.md).

## Active Technologies

- **Rust** (edition 2021, ≥1.85) core: tokio, tree-sitter + tree-sitter-md (incremental editor highlight), comrak (preview HTML), syntect (code highlight), nucleo (Quick Open), notify + notify-debouncer-full (watching), tempfile (atomic writes), reqwest (SSE AI client), thiserror.
- **UniFFI 0.31** boundary → XCFramework (async + foreign-trait callbacks + panic containment).
- **Swift 6 / SwiftUI + AppKit** (Xcode 16.2, macOS 14+): `NSTextView`/TextKit 2 editor, `NSOutlineView` sidebar, `WKWebView` preview (bundled Mermaid + KaTeX), Security framework (Keychain).

## Structure

- `crates/emend-core` — ALL logic, **no FFI dep**, `cargo test` in isolation.
- `crates/emend-ffi` — thin `#[uniffi::export]` shim (the only `uniffi` consumer).
- `swift/EmendCore` — SwiftPM package wrapping the XCFramework + generated bindings.
- `app/Emend` — Xcode macOS app (created during implementation).

## Commands

```bash
just build | test | clippy | fmt-check | check   # check = full pre-push gate
cargo test                                        # core, no Xcode needed
just xcframework                                  # Rust core → EmendCore.xcframework + bindings
swift build && swift test                         # in swift/EmendCore
```

## Project guardrails (do not violate)

- **No panics across FFI** (NFR-003): `emend-core` denies `unwrap_used`/`expect_used`/`panic`; fallible boundary calls return `Result<_, EmendError>`. UniFFI `catch_unwind` contains the rest.
- **FFI ranges are UTF-16 code units** (research §A2) to map onto `NSRange` — never pass UTF-8 offsets across the boundary.
- **Swift owns the text buffer**; per-keystroke edits go to Rust as tiny deltas. Hot path is **synchronous**; async only for AI + search (cancellable via Rust handles, not Swift `Task`).
- **Privacy** (SC-008/FR-035): zero outbound network unless AI is configured AND invoked. Preview WebView blocks remote loads (CSP + nonPersistent + navigation delegate).
- **Autosave is atomic + durable** (FR-009a): tempfile → fsync → rename → fsync dir; debounced (don't fsync per keystroke). Feed post-write `(mtime,len)` to the watcher's self-write suppression so saves don't echo (FR-006a).
- **AI key**: Keychain only, transient to Rust, redacted in the HTTP client — never logged/persisted (NFR-006).
- **Two Markdown engines on purpose**: tree-sitter (editor, incremental, advisory highlight) vs comrak (preview, authoritative HTML). Don't unify them.
- Conventional Commits required (DS-007); `lefthook` runs fmt/clippy/swift-lint pre-commit.
- **MSRV is 1.85** — pin every new dep to a version whose `rust-version ≤ 1.85` (CI enforces with `cargo +1.85 check --all`; e.g. `criterion` stays 0.7 because 0.8 needs 1.86, `ropey` 1.6). Never bump from memory.
- **Generated FFI is git-ignored build output**: `swift/EmendCore/Sources/EmendCoreFFI/` + `xcframework/` are regenerated by `just xcframework` (and in CI) — run it before building the app. The generated `EmendCoreFFI` target compiles in **Swift 5 language mode** (its UniFFI foreign-trait VTables aren't Swift-6 strict-concurrency clean); `EmendCore` + the app stay Swift 6. Bindgen: `cargo install uniffi --features cli` (match the pinned uniffi). Foreign-trait sinks use `#[uniffi::export(with_foreign)]`; the Swift error is `FfiError` (exhaustive mirror of `EmendError`).
- **App unit tests are app-hosted** (`@testable import Emend`); headless logic (bookmark resolution, attribute mapping) is tested there. Security-scoped-bookmark tests use **plain** bookmarks (the test process isn't sandboxed).
- **No XCUITest / GUI automation**: a `bundle.ui-testing` runner can't bootstrap under CI's `CODE_SIGNING_ALLOWED=NO` (CI is GUI/signing-free by design). Test editor flows **headlessly** in the app-hosted bundle by driving `EditorCoordinator` + the pure transforms directly (see `EditorPersistenceTests` for the FR-009 round-trip), never via a launched GUI.
- **SwiftFormat and SwiftLint `--strict` can fight**: `swiftformat` wraps a long multi-line `if`/`guard` condition and drops `{` onto its own line, which SwiftLint `opening_brace` then rejects. Keep conditions one line — precompute a boolean `let`, or use `guard … else { … }` (its brace follows `else`). `nesting` allows types ≤1 level deep; no force-unwraps (`force_unwrapping` is opt-in). Run `swiftformat app swift` then `swiftlint lint --strict` **before** the slow `xcodebuild`.
- **Editor edit model**: list/formatting behaviour is pure `(NSString, NSRange) → Edit` transforms (`SmartLists`/`FormattingCommands`, headless-tested); the `MarkdownTextView` applies each `Edit` via `shouldChangeText`/`replaceCharacters`/`didChangeText` so it registers undo **and** reaches the core through the same storage-delegate delta path as typing. A subclassed `NSTextView` needs an explicit TextKit 2 stack (`NSTextContentStorage`→`NSTextLayoutManager`→`NSTextContainer`); `scrollableTextView()` can't be subclassed.
- **NSOutlineView/AppKit in SwiftUI**: wrap in `NSViewRepresentable` with a `@MainActor` Coordinator that conforms to `NSOutlineViewDataSource`/`Delegate`/`NSMenuDelegate` — those SDK protocols **are** `@MainActor`, so plain conformance compiles (unlike `NSTextStorageDelegate`, which needs `nonisolated`+`assumeIsolated`). Cross-thread Rust→Swift callbacks (e.g. the watcher's `DocObserver`) bridge through a `final class` holding an immutable `@Sendable` closure that hops via `Task { @MainActor in … }`. As Swift files grow, SwiftLint enforces `type_body_length` (250 → move private helpers to a same-file extension) and `file_length` (400 → split the type into its own file).
- **US2 app-state is Swift-side**: the FFI exposes **no getter** for folder-icon/`isFavorite`/`isPinned`, and the core holds favorites/pins/icons/child-order in memory only (no persistence layer yet, despite the data-model's "core-owned store"). So `WorkspaceModel.AppState` (UserDefaults) is the display source of truth + persistence, forwarded to the core via setters and replayed on launch. Location bookmarks (security-scoped) likewise persist Swift-side. A future task should move this core-side + add the getters.
- **US3 Quick Open index seeding + its gap**: the search index is *seeded* by `WorkspaceHandle.reindex_all(max_depth)` (walks each location via core `collect_files`, maps `rel_for`, `Index::rebuild`) — `WorkspaceModel.reindexInBackground()` runs it off-main (`Task.detached`) on launch/add/remove-location (NFR-001). `add_location`/`list_children` do **not** seed; internal file ops (`create_note`/`rename`/`move_node`/`delete`) keep it incremental. **Known gap (FR-017a, deferred):** watcher-driven *external* fs changes refresh the sidebar but do **not** update the core index (no raw `index_insert`/`_remove`/`_rename` FFI yet), so externally-created/deleted files only (dis)appear in Quick Open after the next full reindex. A full reindex per fs event would violate "incremental, no full rescan" — the fix needs those FFI methods wired to `DocObserver`. Quick Open is **files-only** in v1 (folder-in-results deferred; results open into tabs). The Swift `SearchSink` adapter (`QuickOpenSink`) follows the `FsObserver` Sendable-bridge pattern; supersede is double-guarded (FFI cancels prior query + a monotonic `generation` in `QuickOpenModel` drops late batches). `collect_files` canonicalizes paths, so tests/callers must canonicalize the location root before `add_location` or `rel_for` falls back to the basename.
- **US4 preview + PDF export**: one offline `WKWebView` shell (`Resources/preview/template.html` + `bridge.js`, CSP + `nonPersistent` + nav-blocking delegate) drives **two** render entry points — `__emendRender` (live, fire-and-forget) and `__emendRenderForPrint` (async, awaits the `mermaid.run` promise `renderMermaid` now returns). Preview HTML + the syntect theme CSS are **core-owned** (`renderPreviewHtml()`, `previewThemeCss()`); Mermaid/KaTeX are Swift-bundled. **PDF export must use the async `NSPrintOperation.runModal(for:delegate:didRun:contextInfo:)` — NOT synchronous `run()`**: `run()` blocks the main run loop that WebKit needs to deliver web-content-process print IPC → permanent deadlock (it hangs the app *and* the test host). The off-screen print host keeps the WebView in a borderless far-off-screen (`-10000,-10000`) `NSWindow` ordered-front-regardless (else WebKit throttles the occluded view's async JS); both async WebKit steps have a `DispatchQueue.main.asyncAfter` watchdog so an export can never hang forever. Scroll sync (`ScrollSync`, a shared `@StateObject` hub) keys both directions on comrak's 1-based `data-line` anchors; the **active** editor (`isActive: tab.id == activeID`) registers its scroll view, a per-side ~160 ms mute + a JS `applyingScroll` flag guard the feedback loop. Editor scroll is observed via a **selector-based** `boundsDidChangeNotification` observer (a `@objc` method on the `@MainActor` coordinator avoids the Swift-6 `@Sendable` non-Sendable-`self` capture problem). `WKScriptMessageHandler`/`WKNavigationDelegate` **are** `@MainActor` in the Xcode 16.2 SDK → plain conformance. `PreviewExportTests` (multi-page PDF, PDFKit `pageCount > 1`) is headless app-hosted — the `EmendUITests` target named in `tasks.md` does not exist by design.
- **US5 links, embeds, tasks, attachments**: wiki-link resolution matches on the note **stem** (`index.resolve_name`, FR-019a tie-break = same-dir → shallowest → lexicographic), but `SearchHit.name` carries the extension (`Beta.md`) — so `[[` autocomplete must insert `(name as NSString).deletingPathExtension`, else the completed link won't resolve. `[[` autocomplete uses **native NSTextView completion** (`rangeForUserCompletion` is a *property* to override; `completions(forPartialWordRange:…)`; `insertCompletion` closes with `]]`), triggered by `complete(nil)` from `insertText`. **Clickable task checkboxes toggle via a Swift-side `Edit`** (flip the `[ ]`/`[x]` char through the editor `apply` path) — NOT the core `toggleTask` FFI, which would mutate the core buffer and desync Swift's `NSTextStorage` (Swift owns the buffer); `toggleTask` is reserved for non-editor surfaces. Embeds: `OpenDocHandle.render_preview_html_resolving(workspace:)` resolves `![[…]]` via the index + tolerant reads feeding the core cycle/depth-guarded expander (MAX_EMBED_DEPTH=8); the resolver snapshots under the doc lock then **drops all locks before render and before each file read**. Plain `render_preview_html` keeps embeds literal. Image drag-drop stores a collision-safe attachment (`store_attachment`, untitled fallback) and inserts a relative `![](attachments/…)` ref through the Edit path — **a relative image ref does not load in the preview WebView** (its baseURL is the bundle, not the note dir); local-image preview display is a follow-up. Pure transforms (`WikiLink`/`TaskCheckbox`/`ImageDrop`) are headless-tested (`LinkHelpersTests`); the flow is `LinksFlowTests` (not XCUITest). `EditorCoordinator` now lives in its own file (file-length limit). **FR-017a still deferred** (external-fs index maintenance).
- **US6 info sidebar + AI (BYOM)**: the AI client is **split core/FFI** (like US3 search) to keep `emend-core` tokio/reqwest/uniffi-free — the pure SSE line-buffer parser, the FR-036a max-input guard, and the **redacting `ApiKey` newtype** (`Debug`/`Display` → `***`, real bytes only via `expose()`) live in `emend-core/src/ai.rs`; the reqwest streaming (`bytes_stream`, `CancellationToken` + biased `tokio::select!`, **per-chunk** `tokio::time::timeout` — never reqwest's whole-request `.timeout()`) lives in `emend-ffi/src/ai.rs`. `summarize_document` validates locally (blank key/oversized) **before any socket** (SC-008 zero-network), returns an `AiHandle` (`.cancel()`), and streams via the `AiSink` foreign trait (already in `handles.rs`); `test_ai_config` is a sync probe. **reqwest MSRV trap (binding):** reqwest→url→`idna_adapter` 1.2.x pulls `icu_*` 2.x needing rustc **1.86** — pin **`idna_adapter = "=1.1.0"`** in the workspace catalog + Cargo.lock; reqwest is `default-features=false, features=["stream","native-tls"]` (macOS-native TLS, no rustls/ring). The **API key is Keychain-only** (`KeychainStore`, `SecItem` generic password, `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`) — read immediately before a request, passed as a transient `String`, never persisted/logged (NFR-006); `AIConfigStore.isConfigured` (endpoint+model+key) gates the UI. Derived `stats`/`outline` (`OpenDocHandle.stats()`/`outline()`, line-numbered headings) feed `InfoModel`/`InfoSidebarView` (clickable outline → `ScrollSync.scrollToLine`), recomputed off-main + debounced. The Swift `AiSinkBridge` follows the `FsObserver` Sendable-hop; summary supersede is double-guarded (FFI `AiHandle.cancel()` + a `generation` counter). `KeychainStoreTests` does a real round-trip but `XCTSkip`s when the env denies Keychain access (unsigned CI) — never a hang/fail.
- **US7 typography**: core `settings.rs` holds `TypographySettings` (font family, size pt, line height, paragraph spacing) with defaults + idempotent clamping (`set_typography` clamps in-core: size 8–48, line 1–3, paragraph 0–64, non-finite → default); FFI `SettingsHandle.get_typography`/`set_typography`. **No theme field — light/dark follows the system** (FR-039) automatically. Persistence is **Swift-side** (`TypographyModel` → UserDefaults, replayed into the core on launch) per the US2 app-state pattern (core still has no persistence layer). One `Typography` resolver feeds **two appliers**: the editor (`EditorCoordinator` derives base `NSFont` + paragraph style from `typography`; `applyTypography` resets font/paragraph + re-attributes live) and the preview (`PreviewModel.css` = syntect theme **+** `Typography.previewCSS`, injected after `theme.css`; a typography change bumps `version` so the WebView re-injects over unchanged HTML). **PDF export uses `PreviewModel.css`** so the PDF matches the preview's typography. `TypographySettings` is `Equatable` (uniffi Record), enabling the editor's `updateNSView` change-guard and the preview `didSet` version-bump. `TypographySettingsView` (toolbar "Typography") applies live. Headless `TypographyTests` (clamp + persistence round-trip + resolver).
