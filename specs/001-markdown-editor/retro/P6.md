## Packages & Dependencies
<!-- Useful packages discovered, packages to avoid, version issues -->

- US4 introduces the **comrak** preview engine + **syntect** code highlighting. Per the
  "two Markdown engines on purpose" guardrail: tree-sitter is the editor's incremental
  advisory highlighter; comrak is the preview's authoritative HTML. **Do not unify them.**
- MSRV 1.85: pin every new dep to a `rust-version ≤ 1.85` (CI runs `cargo +1.85 check --all`);
  verify on crates.io before pinning. Confirm comrak + syntect versions' MSRV.
- T083a: generate & vendor a **binary** syntect `SyntaxSet`/`ThemeSet` dump (30-lang v1 set,
  research §D) into `crates/emend-core/assets/`; lazy-load ≤23 ms, never raw-YAML on the hot
  path (research §B6).

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- **Privacy (SC-008/FR-035) is binding for the preview:** the `WKWebView` must block all
  remote loads — offline CSP + `nonPersistent` data store + a navigation-blocking delegate.
  Mermaid + KaTeX are **bundled** (vendored under `app/Emend/Emend/Resources/preview`, already
  a folder reference in `project.yml`); `template.html` resolves them via relative paths, so
  point the WebView `baseURL` at that folder. Zero outbound network from preview, ever.
- Scroll sync (T088) uses `data-line` anchors emitted by comrak (T084); guard the
  bidirectional editor↔preview sync against feedback loops.
- PDF export (T089) via `NSPrintOperation` on an **offscreen** `WKWebView` with `@media print`.
