# Phase 8 Retrospective: US6 — Info sidebar + AI summary

Scope: live info sidebar (word/char/reading-time stats, N-of-M task completion,
clickable outline → scroll), and an on-demand BYOM AI summary over an
OpenAI-compatible endpoint. **Zero network unless AI is configured AND invoked.**

## What Worked Well
<!-- Effective patterns, tools, approaches -->

- **The core/FFI split held cleanly** (mirrors US3 search): the pure SSE line-buffer
  parser, FR-036a input guard, and the redacting `ApiKey` newtype live in
  `emend-core/src/ai.rs` (tokio/reqwest-free, unit-tested in isolation); the
  reqwest+tokio streaming (`bytes_stream`, `CancellationToken` + biased
  `tokio::select!`, per-chunk inactivity `timeout`) lives in `emend-ffi`. Core stayed
  free of tokio/uniffi/reqwest (proven by `cargo tree -i`).
- `AiSink`/`DocObserver` foreign traits **already existed** in `emend-ffi/handles.rs`
  from the Foundational phase — the AI/derived work wired them rather than redefining,
  and the Swift `AiSinkBridge` reused the established `FsObserver` Sendable-hop pattern.
- Validating locally (blank key / oversized input) **before constructing any request**
  makes "zero network unless configured + invoked" (SC-008) structural, not incidental.
- The Keychain round-trip test passes locally and **skips** (doesn't fail) where the
  environment denies access — so CI stays green without faking it.

## What Didn't Work
<!-- Problems, issues, limitations encountered -->

- **MSRV 1.85 vs reqwest's transitive `idna`**: reqwest → `url` → `idna_adapter` 1.2.x
  pulls `icu_*` 2.x which require rustc **1.86**, breaking `cargo +1.85.0 check`. The
  newest reqwest itself is fine; the dependency tree wasn't.
- macOS Keychain from an unsigned/headless test process can return
  `errSecMissingEntitlement` rather than working — a real test can't assume access.

## Workarounds & Solutions
<!-- Bug fixes, non-obvious solutions, clever workarounds -->

- Pinned **`idna_adapter = "=1.1.0"`** in the workspace catalog + lockfile (its
  `unicode-*` backend is < 1.85), so the whole reqwest tree builds under 1.85. reqwest
  `0.13` with `default-features = false, features = ["stream", "native-tls"]` — the
  macOS-native TLS backend avoids the rustls/ring/aws-lc MSRV churn entirely.
- `KeychainStore.save` surfaces the `OSStatus`; `KeychainStoreTests` does the real
  round-trip but `throw XCTSkip` when save returns an access error (no hang — headless
  Keychain returns an error, it doesn't prompt).
- Per-chunk `tokio::time::timeout` (NOT reqwest's whole-request `.timeout()`, which
  fires mid-stream); cancel is raced in the same `select!` so a pre-headers cancel
  resolves promptly; exactly one terminal sink callback per stream.

## Packages & Dependencies
<!-- Useful packages discovered, packages to avoid, version issues -->

- **`reqwest` (research §B5)** is a NEW dep and goes in **`emend-ffi` only** — it pulls
  tokio/hyper, and `emend-core` MUST stay tokio-free + reqwest-free (binding guardrail;
  CI runs `cargo tree -p emend-core -i tokio`/`-i uniffi` and both must be absent).
  **MSRV 1.85 is the live risk**: reqwest + hyper + its TLS backend must all have
  `rust-version ≤ 1.85` — verify with `cargo +1.85.0 check --all` before trusting it;
  pin an older `reqwest` if the latest needs >1.85. Prefer the macOS-native TLS backend
  to avoid ring/rustls MSRV churn. Feature `stream` for `bytes_stream()`.

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- **Core/FFI split for AI (mirrors US3 search):** the *pure, tokio-free* pieces live in
  `emend-core/src/ai.rs` — SSE line-buffer parser (`data:` accumulation across chunks,
  `[DONE]`, comment/heartbeat lines, CRLF/LF), max-input-size validation (FR-036a), and
  the **redacting key newtype** (`Debug`/`Display` → `***`, never a tracing field —
  NFR-006). The *reqwest + tokio* orchestration (`bytes_stream()`,
  `CancellationToken` + `tokio::select!`, per-chunk inactivity `timeout`) lives in
  **`emend-ffi`**, feeding chunks through the core parser. Tests T109 (SSE parse) and
  T110 (privacy: key redaction, no-send-when-unconfigured) are pure → `emend-core`.
- **Privacy (SC-008/FR-035/NFR-006) is binding**: the key is Keychain-only (Swift owns
  custody, research §C5 — `SecItem` `kSecClassGenericPassword`,
  `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`), read immediately before each
  request and passed across FFI as a transient `String`; never persisted Rust-side,
  never logged; only set on the `Authorization` header. No request is made unless a
  config exists AND the user invokes summarize.
- Derived stats/outline/links are pure `emend-core` (`derived.rs`, already holds US5
  link/task extraction); the live push to the info sidebar reuses the `DocObserver`
  callback pattern. Outline rows map to source lines (reuse the US4 `data-line` idea)
  so clicking scrolls the editor.
- KeychainStore tests use plain items (the test process isn't sandboxed); the AI summary
  view streams via an `AiSink` foreign-trait bridge (same Sendable-closure hop as
  `FsObserver`/`QuickOpenSink`), supersede-on-reinvoke guarded.
