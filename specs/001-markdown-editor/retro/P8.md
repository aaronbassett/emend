## Packages & Dependencies
<!-- Useful packages discovered, packages to avoid, version issues -->

- **`reqwest` (research ¬ßB5)** is a NEW dep and goes in **`emend-ffi` only** ‚Äî it pulls
  tokio/hyper, and `emend-core` MUST stay tokio-free + reqwest-free (binding guardrail;
  CI runs `cargo tree -p emend-core -i tokio`/`-i uniffi` and both must be absent).
  **MSRV 1.85 is the live risk**: reqwest + hyper + its TLS backend must all have
  `rust-version ‚â§ 1.85` ‚Äî verify with `cargo +1.85.0 check --all` before trusting it;
  pin an older `reqwest` if the latest needs >1.85. Prefer the macOS-native TLS backend
  to avoid ring/rustls MSRV churn. Feature `stream` for `bytes_stream()`.

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- **Core/FFI split for AI (mirrors US3 search):** the *pure, tokio-free* pieces live in
  `emend-core/src/ai.rs` ‚Äî SSE line-buffer parser (`data:` accumulation across chunks,
  `[DONE]`, comment/heartbeat lines, CRLF/LF), max-input-size validation (FR-036a), and
  the **redacting key newtype** (`Debug`/`Display` ‚Üí `***`, never a tracing field ‚Äî
  NFR-006). The *reqwest + tokio* orchestration (`bytes_stream()`,
  `CancellationToken` + `tokio::select!`, per-chunk inactivity `timeout`) lives in
  **`emend-ffi`**, feeding chunks through the core parser. Tests T109 (SSE parse) and
  T110 (privacy: key redaction, no-send-when-unconfigured) are pure ‚Üí `emend-core`.
- **Privacy (SC-008/FR-035/NFR-006) is binding**: the key is Keychain-only (Swift owns
  custody, research ¬ßC5 ‚Äî `SecItem` `kSecClassGenericPassword`,
  `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`), read immediately before each
  request and passed across FFI as a transient `String`; never persisted Rust-side,
  never logged; only set on the `Authorization` header. No request is made unless a
  config exists AND the user invokes summarize.
- Derived stats/outline/links are pure `emend-core` (`derived.rs`, already holds US5
  link/task extraction); the live push to the info sidebar reuses the `DocObserver`
  callback pattern. Outline rows map to source lines (reuse the US4 `data-line` idea)
  so clicking scrolls the editor.
- KeychainStore tests use plain items (the test process isn't sandboxed); the AI summary
  view streams via an `AiSink` foreign-trait bridge (same Sendable-closure hop as
  `FsObserver`/`QuickOpenSink`), supersede-on-reinvoke guarded.
