# Phase 3 Retrospective: US1 — Live, distraction-free editor (MVP)

## What Worked Well
<!-- Effective patterns, tools, approaches -->

- **Pure transforms + headless tests for editor behaviour.** Smart lists (T045) and
  formatting commands (T046) are pure functions of `(NSString, NSRange) → Edit`,
  unit-tested in the app-hosted bundle with no window. TDD was a clean fit: write
  the string-round-trip assertion, watch it fail (stubs return `nil`/identity),
  implement. The thin `NSTextView` glue stays untested per Constitution VII.
- **Routing edits through the existing storage-delegate path.** `MarkdownTextView`
  applies every list/formatting `Edit` via `shouldChangeText`/`replaceCharacters`/
  `didChangeText`, so they register undo AND flow to the Rust core as ordinary
  deltas through the same `EditorCoordinator` path as typing — no new core wiring.
- **Headless integration test for persistence (T049).** Driving the real
  `EditorCoordinator` + `AutosaveController` over an open doc, then asserting the
  on-disk + `readFileAt` round-trip, verifies FR-009 end-to-end without GUI
  automation.

## What Didn't Work
<!-- Problems, issues, limitations encountered -->

- **XCUITest is incompatible with this project's CI.** A `bundle.ui-testing`
  target's runner (XCTRunner) cannot bootstrap under `CODE_SIGNING_ALLOWED=NO`
  ("Test crashed with signal kill before starting test execution"), which is the
  exact flag CI (and `just app-test`) use. The project's CI is GUI/signing-free by
  design (ci.yml comment). **T049 was reimplemented as a headless app-hosted
  integration test** (`EditorPersistenceTests`) instead of `EditorFlowTests`. If a
  true GUI smoke test is ever wanted, it needs a separately-signed runner + a
  dedicated CI job — out of scope for the MVP.
- **SwiftFormat ↔ SwiftLint brace conflict.** `swiftformat` wraps long multi-line
  `if`/`guard` conditions and drops the `{` onto its own line; `swiftlint`'s
  `opening_brace` (strict) then rejects it. They fight to a standstill on wrapped
  conditions.

## Workarounds & Solutions
<!-- Bug fixes, non-obvious solutions, clever workarounds -->

- **Brace conflict:** keep the `{` on the same line by not letting conditions wrap
  — precompute a boolean `let` (e.g. `let spansMarkers = …`) and `if spansMarkers {`,
  or use `guard … else { return }` (its brace follows `else`, so it's exempt), or
  split nested conditions. Don't write multi-line bare `if` conditions.
- **`nesting` (strict):** types may nest at most one level. Moved `ListItem.Kind`
  out to a sibling `private enum ListKind` of the enclosing namespace enum.
- **`Ambiguous use of 'greatestFiniteMagnitude'`** in `NSSize(width:height:)` — the
  `CGFloat`/`Double`/`Int` overloads make the bare member ambiguous. Bind a
  `let maxDimension = CGFloat.greatestFiniteMagnitude` first.
- **Swift 6 + XCTest:** XCUITest APIs and any test touching `@MainActor` state must
  run on the main actor — annotate the test class `@MainActor`. `setUp`/`tearDown`
  overrides are nonisolated, so don't have them touch `@MainActor` stored
  properties; create per-test temp state inside the (`@MainActor`) test method.

## Packages & Dependencies
<!-- Useful packages discovered, packages to avoid, version issues -->

- Carried from P2: verify a new dep's `rust-version ≤ 1.85` before pinning (CI runs `cargo +1.85 check --all`). tree-sitter + tree-sitter-md become the first heavy core deps to wire here.
- US1 added no new dependencies (smart lists / formatting are pure Swift; the

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- The document FFI (T039) wraps `emend_core::document::Document` behind a `Mutex` uniffi object; `push_edit` stays synchronous (hot path) and may stay `Result`-returning at the boundary (a malformed delta is worth surfacing). `Document` exposes `utf16_to_line_col`/`line_col_to_utf16` + `len_utf16` for viewport→line mapping; a `utf16↔char`/`char_to_byte` bridge is still needed so tree-sitter (byte/char axis) spans translate to UTF-16 `U16Range` for the boundary.
- Highlight spans are advisory (tree-sitter editor engine), distinct from comrak preview — do not unify (Constitution).
- **Editor `Edit` value type** `(range, replacement, selectionAfter)` is the unit of
  every list/formatting transform; the view applies it uniformly. Keeping the
  transform pure (string in, `Edit` out) is what makes the behaviour testable.
- **TextKit 2 + an `NSTextView` subclass:** `NSTextView.scrollableTextView()` can't
  be subclassed, so build the stack explicitly — `NSTextContentStorage` →
  `NSTextLayoutManager` → `NSTextContainer` → `MarkdownTextView(frame:textContainer:)`.
  `textView.textStorage` still bridges, so the coordinator's delegate path is
  unchanged. Override `insertNewline/insertTab/insertBacktab` for list keys and
  `performKeyEquivalent` for ⌘B/⌘I/⌘K/⌘⇧T.

## For Next Time
<!-- Improvement suggestions, lessons learned, things to try differently -->

- When a task prescribes a tool (e.g. XCUITest), check it against the project's CI
  constraints *first* — here a headless test was both spec-faithful (FR-009) and
  CI-compatible, where the prescribed GUI test was neither.
- Run `swiftformat app swift` (apply) then `swiftlint --strict` locally before the
  slow `xcodebuild` round-trip; the two-tool conflicts surface in <1s and save a
  multi-minute build.
- Smart-list scope deliberately bounded for the MVP: indent/outdent act on the
  caret line only (not multi-line selections) and don't re-base ordered numbers on
  nesting changes; renumber handles the contiguous same-indent block. Revisit if
  US-level polish calls for it.
