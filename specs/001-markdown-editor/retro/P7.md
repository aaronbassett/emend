# Phase 7 Retrospective: US5 — Link and connect notes

Scope: `[[wiki links]]` with autocomplete + deterministic resolution, `![[embeds]]`
(cycle/depth-guarded), clickable task checkboxes, `==highlight==`, drag-drop images.
Depends on US2 (index) + US4 (preview).

## What Worked Well
<!-- Effective patterns, tools, approaches -->

- **Native NSTextView completion for `[[`** — overriding `rangeForUserCompletion`
  (a *property*, not a method) + `completions(forPartialWordRange:…)` +
  `insertCompletion(…)` gives a system completion dropdown that integrates with undo
  and the storage-delegate delta path for free, far less code than a custom panel.
  Triggered live by calling `complete(nil)` from `insertText` while inside an open link.
- **Pure transforms headless-tested**: `WikiLink`/`TaskCheckbox`/`ImageDrop` are pure
  `NSString → range/edit` functions (`LinkHelpersTests`, 11), and the end-to-end flow
  (`LinksFlowTests`, 5) drives the real `WorkspaceHandle`/`OpenDocHandle` + coordinator —
  no XCUITest. The flow test caught a real resolution bug (below).
- **Core embed expander + FFI resolver closure**: the core `expand_embeds` owns
  recursion + cycle/depth (MAX_EMBED_DEPTH=8); the FFI passes a resolver that snapshots
  `(text, note path)` under the doc lock, drops it, then resolves each target via the
  index (lock dropped before the tolerant on-disk read) — no lock held across render/IO.

## What Didn't Work
<!-- Problems, issues, limitations encountered -->

- **`SearchHit.name` carries the file extension** (`Beta.md`), but wiki-link resolution
  matches on the stem (`Beta`). Autocomplete first inserted `[[Beta.md]]`, which then
  failed to resolve and rendered as a (red) unresolved link. The flow test caught it.
- The **SwiftFormat ↔ SwiftLint brace fight** struck again: multi-line `if a, b {`
  bracket-pair conditions get the `{` dropped to its own line → `opening_brace` rejects.
- `MarkdownEditorView.swift` blew past the 400-line `file_length` limit once the
  wiki-link/checkbox/drag-drop overrides landed.

## Workarounds & Solutions
<!-- Bug fixes, non-obvious solutions, clever workarounds -->

- Autocomplete inserts the **stem** (`(name as NSString).deletingPathExtension`) so the
  completed `[[Beta]]` resolves; `LinksFlowTests.testCoordinatorLinkServices` pins it.
- Bracket-pair checks moved into single-expression helpers (`doubledEndingAt`/
  `doubledStartingAt`) so conditions stay one line; loops use `guard … else { continue }`.
- Split `EditorCoordinator` into its own file (`EditorCoordinator.swift`).
- **Checkbox toggle is a Swift-side `Edit`** (flip the `[ ]`/`[x]` char via the editor's
  `apply` path), NOT the core `toggleTask` FFI — Swift owns the buffer, so a core-side
  mutation would desync the `NSTextStorage`. The `toggleTask` FFI is left for non-editor
  surfaces (preview / info pane).

## Packages & Dependencies
<!-- Useful packages discovered, packages to avoid, version issues -->

- No new Rust deps expected — link/task extraction reuses the existing tree-sitter /
  comrak parses; embeds reuse the comrak preview. MSRV 1.85 still binding; `emend-core`
  stays uniffi/tokio-free; no panics across FFI.

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- Wiki-link resolution (FR-019a): deterministic for duplicate basenames (define + test the
  tie-break, e.g. shortest/lexicographically-first path); a rename leaves old links
  unresolved (no auto-rewrite in v1). Resolution draws on the US2 index.
- Embed resolution (FR-021a): `![[…]]` inlines a note's rendered content in the preview with
  a **cycle + max-depth guard** so `A embeds B embeds A` terminates.
- Attachments (FR-013a): `store_attachment` is collision-safe with an untitled fallback.
- Tasks (FR-014): clickable checkbox toggles `[ ]`/`[x]` through the same `Edit`-applying
  path as typing so it registers undo and reaches the core (mirror `SmartLists`/`FormattingCommands`).
- T104 link-flow test is specced under `EmendUITests`; implement it **headless app-hosted**
  in `EmendTests` like the other flows (the `EmendUITests` target doesn't exist by design —
  see the US4 PreviewExportTests precedent).

## For Next Time
<!-- Improvement suggestions, lessons learned, things to try differently -->

- **Carried forward (FR-017a, still deferred):** external-change incremental index
  maintenance — internal file ops keep the index incrementally, but watcher-driven external
  fs changes still need raw `index_insert`/`_remove`/`_rename` FFI wired to `DocObserver`, so
  externally-created files appear in `[[` autocomplete / Quick Open only after a reindex.
  Not blocking US5's tasks (the index is seeded on launch + maintained for in-app ops); close
  it before relying on live external-file linking.
- US4 lesson to honour here: do NOT add a live-`WKWebView` GUI assertion that only the
  headless CI can flake on; keep embed-rendering tests at the deterministic core/HTML layer.
