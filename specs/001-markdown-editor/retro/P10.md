# Phase 10 Retrospective: Polish & Cross-Cutting Concerns

Scope: quality/performance/verification across all seven user stories — perf benches
vs budgets, bounded-memory and large-file tests, accessibility identifiers, a security
review, and end-to-end quickstart validation. No new user-facing features.

## What Worked Well
<!-- Effective patterns, tools, approaches -->

- **The "tracked, non-blocking" perf posture (Principle IV) paid off.** Recording real
  Criterion numbers surfaced an honest, useful finding — Quick Open (SC-004) has ~40×
  headroom, but the `tree-sitter-md` split parser makes both initial parse (SC-002) and
  per-keystroke reparse (SC-003) O(document size), so the 1 MiB worst case is over
  budget while typical (≤64 KB) notes are comfortably under. Because budgets don't gate
  CI, this is documented and queued as a follow-up rather than a release blocker.
- **Weak-reference release tests are a clean, deterministic NFR-005 check.** Driving
  `TabModel` directly and asserting the Rust-backed `OpenDocHandle` + `AutosaveController`
  deallocate on close/reload needs no GUI and no instrumentation — ARC nils the weak
  refs the instant the last strong ref drops, so a leak fails the test immediately.
- **Splitting the Rust work to one `rust-dev` agent (FR-027a test + SC-002 bench) while
  doing Swift directly** kept the core-lint discipline (no `unwrap`/`expect`/`panic`
  outside `#[cfg(test)]`, `black_box` in benches) intact and let the two streams run in
  parallel without stepping on each other.

## What Didn't Work
<!-- Problems, issues, limitations encountered -->

- **The agent didn't run `cargo fmt`**, so the first `just check` failed on `fmt-check`
  (a multi-line `match` arm and an `assert_eq!` rustfmt wanted reflowed). Cheap fix
  (`cargo fmt`), but a reminder to always re-run the *full* gate locally before trusting
  agent output — which is exactly the handover protocol.
- **A genuine functional gap was hiding in plain sight:** the App Sandbox entitlements
  never included `com.apple.security.network.client`, so the shipped US6 AI feature
  would have failed to connect in a sandboxed build (the `reqwest` client connects at
  the socket layer, which the sandbox denies without that key). Found only because the
  security review enumerated the outbound path end-to-end.

## Workarounds & Solutions
<!-- Bug fixes, non-obvious solutions, clever workarounds -->

- **Added `com.apple.security.network.client`** with a comment making the privacy model
  explicit: the entitlement grants *capability*, but SC-008 zero-network is enforced in
  *code* (`summarize_document`/`test_ai_config` refuse before any socket when the key is
  blank), and the preview WebView still can't reach the network (CSP `connect-src 'none'`
  + `nonPersistent` + nav-blocking delegate). Least privilege preserved: no
  `network.server`, no ATS `NSAllowsArbitraryLoads`.
- **ATS doesn't apply to the AI client** because `reqwest` uses raw tokio TCP +
  Security.framework TLS, not CFNetwork/`URLSession` — so a BYOM `http://localhost`
  endpoint (local model) works without a cleartext exception, and we keep the
  default-deny posture for any future CFNetwork use. Documented in the review.
- **SC-002 has no pure "first paint" number**, so the `open_doc` bench measures the
  core open + initial parse (the dominant pre-paint compute) and the report notes that
  the editor seeds raw text into TextKit *before* the advisory highlight, so the
  user-visible "first content" milestone precedes the measured parse cost.

## Packages & Dependencies
<!-- Useful packages discovered, packages to avoid, version issues -->

- No new dependencies. The new test/bench reuse `tempfile`/`criterion`/`emend-core`
  already in the workspace; MSRV stays 1.85; `emend-core` stays uniffi/tokio/reqwest-free.

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- **`.xcodeproj` is generated by xcodegen from `app/Emend/project.yml` and git-ignored;
  the `EmendTests` target folder-globs `EmendTests/`.** New test files are picked up
  automatically on `just xcodeproj` / `just app-test` — no manual `pbxproj` surgery
  needed (a manual edit is harmless but gets overwritten on regeneration). The source of
  truth for target membership is `project.yml`, not the generated project.
- **Accessibility identifiers** (`editor.textView`, `sidebar.outline`,
  `quickOpen.searchField`/`.results`/`.result.<name>`) are set even though there's no
  XCUITest target by design (Constitution VII) — they cost nothing, aid VoiceOver, and
  leave the door open if a signed UI-test lane is ever added outside CI.
- **Quickstart drift corrections** worth remembering: the bindgen install is
  `cargo install uniffi --version 0.31.1 --features cli --bin uniffi-bindgen-swift`
  (not a standalone `uniffi-bindgen-swift` crate), and the testing story is *headless
  app-hosted*, never XCUITest.

## For Next Time
<!-- Improvement suggestions, lessons learned, things to try differently -->

- **Decouple the advisory highlight from the synchronous keystroke path for large docs**
  (debounce/background the reparse; optimistic viewport attributing) — the single most
  impactful perf follow-up, directly addressing the 1 MiB SC-003/SC-002 numbers without
  touching the O(log n) buffer hot path. Pairs naturally with a size threshold above
  which highlighting is throttled, bounded by the FR-027a 5 MB read-only cap.
- **FR-017a (external-fs incremental index maintenance) remains the one deferred
  functional gap** — watcher-driven external file changes refresh the sidebar but not
  the Quick Open index until the next full reindex. Needs `index_insert/_remove/_rename`
  FFI wired to `DocObserver`. Carried past v1; the cleanest first item for a follow-up.
- Consider an app-hosted `XCTMetric` keystroke-latency test if a windowed CI lane
  becomes available, to capture the TextKit-inclusive latency the core benches can't.
