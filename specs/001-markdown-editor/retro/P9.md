## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- **Core holds `TypographySettings` in-memory with defaults + clamping**; the FFI
  exposes `get_typography`/`set_typography` (contract §8). Per the established US2
  app-state pattern, **persistence is Swift-side** (UserDefaults) and replayed into
  the core on launch — the data-model envisions a core-owned app-support store, but
  the core still has no persistence layer.
- Typography applies to **two render surfaces**: the editor (`NSTextView` base font +
  line spacing via paragraph style / `defaultParagraphStyle`, re-attributed) and the
  preview (CSS variables injected into the WKWebView — extend the `bridge.js`
  theme-injection used for the syntect CSS). One settings source, two appliers.
- **Light/dark follows the system automatically** (SwiftUI/AppKit `NSAppearance`); the
  preview CSS already uses `prefers-color-scheme`. v1 has no manual theme toggle — so
  `TypographySettings` carries font/size/spacing, not a theme field.
